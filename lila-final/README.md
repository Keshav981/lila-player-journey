# LILA BLACK вЂ” Player Journey Visualizer

A browser-based telemetry visualization tool for LILA Games' Level Design team.  
Turns raw `.nakama-0` parquet files into an interactive map showing player paths, kill zones, loot density, and storm deaths.

**Live Demo:** [Deploy to Vercel вЂ” see setup below]

---

## Quick Start

### Step 1 вЂ” Convert the data (run once on your machine)

```bash
pip install pandas pyarrow
python scripts/convert_data.py --input /path/to/player_data --output public/data.json
```

### Step 2 вЂ” Run locally

```bash
npm install
npm start
```

Opens at `http://localhost:3000`

### Step 3 вЂ” Deploy to Vercel

```bash
npm install -g vercel
npm run build
vercel --prod
```

Or connect your GitHub repo to [vercel.com](https://vercel.com) and it auto-deploys on every push.

## Features

| Feature | Description |
|---|---|
| **Map selector** | Switch between GrandRift, AmbroseValley, Lockdown |
| **Date & match filter** | Drill into a specific day or individual match |
| **Human vs Bot toggle** | Isolate human player paths or bot paths |
| **Event type filters** | Show/hide movement, kills, deaths, storm, loot |
| **Heatmap overlays** | Kill zones, death zones, loot density, storm deaths, traffic |
| **Timeline playback** | Watch a match unfold with scrubable playhead |
| **Hover tooltips** | Click any event point to see player ID, coordinates, timestamp |

## Notes

- If `data.json` is missing, the tool runs in **Demo Mode** with synthetic data so you can see the UI
- Coordinate mapping auto-fits to the data bounds вЂ” no manual calibration needed
- Bot detection: numeric user IDs = humans, UUID user IDs = bots
